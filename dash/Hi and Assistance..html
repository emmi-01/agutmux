python
from flask import Flask, request, jsonify
from flask_cors import CORS
from pymongo import MongoClient

app = Flask(__name__)
CORS(app)
string = "mongodb+srv://test:test@cluster0.dnxuorp.mongodb.net/?retryWrites=true&w=majority"

client = MongoClient(string)
db = client["tour_booking_db"]
tour_bookings_collection = db["tour_bookings"]

# Route to fetch tour bookings
@app.route('/tour-bookings')
def tour_bookings():
    # Parse query parameters
    page = int(request.args.get('page', 1))
    page_size = int(request.args.get('page_size', 10))
    sort_by = request.args.get('sort_by', 'timestamp')
    sort_order = int(request.args.get('sort_order', -1))

    # Compute skip and limit values for pagination
    skip = (page - 1) * page_size
    limit = page_size

    # Query the database for tour bookings
    query = tour_bookings_collection.find().sort(sort_by, sort_order).skip(skip).limit(limit)
    tour_bookings = list(query)

    # Construct response
    response = {
        'tour_bookings': tour_bookings,
        'page': page,
        'page_size': page_size,
        'has_next': len(tour_bookings) == page_size,
    }

    return jsonify(response)

if __name__ == '__main__':
    app.run(debug=True)

The new route is defined at /tour-bookings and accepts the following query parameters:

  • page: the current page number (default: 1)
  • page_size: the number of results to return per page (default: 10)
  • sort_by: the field to sort by (default: 'timestamp')
  • sort_order: the sort order (1 for ascending, -1 for descending; default: -1)

The route fetches tour booking data from the MongoDB database using the find() method and applies sorting and pagination using the sort(), skip(), and limit() methods. The results are returned as a JSON response with the following structure:

so what's document in mongo

In MongoDB, a document is a basic unit of data. It is roughly equivalent to a row in a relational database management system (RDBMS). Each document is stored as a JSON (JavaScript Object Notation) object, which means that it can contain nested fields and arrays.

Documents are stored in collections, which are the equivalent of tables in a RDBMS. A collection is a group of related documents that have a set of common fields. Unlike in a RDBMS, the documents in a collection can have different fields and there is no schema that defines the structure of the documents in advance. Instead, the fields are defined dynamically as data is inserted into the collection.

MongoDB is a document-oriented database, which means that it is designed to store and manipulate documents directly, rather than storing data in tables with rows and columns as in a relational database.
